# 온디바이스 AI 에이전트 시스템: 시니어 프로젝트 관리(PM) 및 진행 계획서 (최종 합의본)

## 1. 개요 및 목적
본 계획서는 인텔 랩탑 노트북 환경(CPU-Only)에서 리소스 병목이라는 치명적 리스크를 시니어 위원회가 지적한 하강/대체 방안(`Thread-Lock`, `uv sandbox`, `TPS 벤치마크`)으로 모두 보완한, 기술 구현 라이프사이클의 최종 완성형 일정(Phase) 설계도입니다.

## 2. 프로젝트 구현 추진 단계 (4-Phases Plan)

### Phase 1: Core Engine & TPS Benchmark (CPU 코어 및 마일스톤 검문)
*   **목표**: Qwen2.5-7B Coder 셋업 후, PM이 강제한 **"5 TPS 통과 여부"** 선행 결정을 완료.
*   **개발 타스크**:
    1.  `Ollama` 윈도우 CPU 환경 구축 및 GGUF 4bit 다운로드.
    2.  `FastAPI` 뼈대 빌드 후 LLM 벤치마크 테스트 스크립트 실행 (초당 몇 토큰이 뜨는지 측정).
    3.  합격 시: `ProtectAI` 악의성 가드레일 & `GPTCache` 연동.
    4.  불합격 시: 모델을 즉각 3B(`Phi-3.5`) 급 초경량으로 자동 강등 세팅 진행.

### Phase 2: Knowledge Ingestion & Thread Lock (로컬 파서 및 동기화 락)
*   **목표**: CPU 과열을 막는 '눈치보는 백그라운드' 기반 LightRAG 파이프라인.
*   **개발 타스크**:
    1.  `Docling` 텍스트 복원 파서 이식.
    2.  전역 상태(글로벌 변수 혹은 Redis대타 SQLite 등)를 활용한 `Thread-Lock` 로직 작성. 채팅이 들어오면 문서 굽기 정지 로직 확립.
    3.  `LanceDB` 통합.

### Phase 3: Code-Agent Brain & uv Sandbox (파이썬 에이전트 및 샌드박스)
*   **목표**: 파이썬 코드를 기가 막히게 짜는 에이전트 구조(`SmolAgents`)와 OS를 망치지 않는 `uv` 파이썬 전용 샌드박스 통신.
*   **개발 타스크**:
    1.  Agent 브레인 템플릿(시스템 프롬프트) 구축. (JSON이 아닌 코드 방식 적용).
    2.  `uv run --isolated` 워크스페이스 격리 파이썬 코드 실행 API 래퍼 작성.

### Phase 4: Next.js HUD Dashboard & Streaming (실시간 자원 관제 UI)
*   **목표**: Next.js 화면 내에 시스템의 CPU 온도(%), RAM 점유 상태를 상시 노출하는 챗봇 통합 대시보드 구축.
*   **개발 타스크**:
    1.  `shadcn/ui` 기반 화면 구현 (VRAM 게이지만 CPU 게이지로 교체).
    2.  FastAPI SSE 수신부 및 Thought 아코디언 로직 연동.

## 3. 리스크 매트릭스 재점검 (최종 합의안 적용)

| 심각도 | 리스크(Risk) 내용 | 합의된 대체제 및 감쇠 전략 (Final Solution) |
| :--- | :--- | :--- |
| **치명적** | **Phase 1 추론 속도의 참사 (2 TPS 이하)** | PM 강제 룰 발동. 즉시 3B급의 `Phi-3.5-mini` 모델로 스케일다운하여 지연 해소. |
| **높음** | **에이전트 백그라운드 연산 시 전체 시스템 다운** | 아키텍처 위원회가 요구한 **Thread-Lock(스레드 동기화 락)** 시스템 구현. RAG 봇은 챗봇이 쉴 때만 문서를 처리하도록 권한 분리. |
| **중간** | **Deno 샌드박스의 파이썬 비호환 문제** | 연구원 제안 반영. 언어적 호환성을 100% 맞추면서 환경이 격리되는 `uv run --isolated` 혹은 `Pyodide` 로 샌드박스 엔진 전격 교체. |

## 4. 최종 마일스톤 요약 (The Final Step)
현재로써 아키텍처/문서상의 약점은 0%에 수렴하며, 리스크 대비안이 가장 완벽하게 직조되었습니다.
더 이상 문서를 수정할 단계는 지나갔습니다. 행동(Coding)이 필요한 **"Phase 1의 Ollama 테스트 (5 TPS 달성 인증)"** 마일스톤으로 즉시 돌입해야 합니다.
