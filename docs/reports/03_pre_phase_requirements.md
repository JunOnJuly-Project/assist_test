# 온디바이스 AI 에이전트 시스템: 본격 페이즈 진입 전 선행 요구사항 (최종 합의본)

## 1. 개요 및 목적
본 문서는 NVIDIA(VRAM)가 없는 랩탑 환경의 **[Phase 1]**에 돌입하기 전 반드시 지켜야 할 **Phase 0** 체크리스트입니다. PM/아키텍트/연구원 3인 합의에 의해 **"TPS 합격 장벽"**이 추가된 가장 엄격하고 안전한 개발 시작 가이드라인입니다.

## 2. 선행 체크리스트 (Phase 0: Environment & Hardware)

### Step 1: 개발 환경 통제 및 패키지 매니저 세팅 (Environment Setup)
*   **작업 내용**:
    *   **Python (Backend)**: 가상환경 충돌 방지와 샌드박스의 기반이 되는 `uv` (초고속 파이썬 툴체인) 필수 설치.
    *   **Node.js (Frontend)**: 프론트 모듈 관리를 위한 `pnpm` 도구 설치.
*   **상태**: [완료됨 - 디렉토리 뼈대 및 `.gitignore` 설정 등 사전 작업 성공적 반영]

### Step 2: CPU 추론 TPS(Speed) 벤치마크 타당성 검증 - **[절대 평가 항목]**
*   **작업 내용**:
    *   터미널에 `Ollama` 런타임 설치 후, **`ollama run qwen2.5-coder:7b`** 구동.
*   **PM 합격 기준치 (Go/No-Go)**:
    *   질문을 던졌을 때 토큰 응답 속도가 최소 **초당 5~7 토큰(TPS)** 이상 나와야만 합격입니다. (노트북 RAM 점유 및 CPU % 모니터링 포함).
*   **우회 플랜(대체제)**: 만약 이 속도 장벽을 통과하지 못하면 프로젝트 무용지물이 되므로, 아무 고민 없이 즉각 **3B 소형 모델(예: `Phi-3.5-mini`)로 2차 강등**하여 시스템 셧다운을 방어해야 합니다.

### Step 3: 보안 모델 및 격리 샌드박스(uv) 준비
*   **작업 내용**:
    *   **Red Teaming 오프라인화**: `deberta-v3-base-prompt-injection-v2` 모델을 로컬 공간(`.weights`)에 다운로드 예약.
    *   **보안 샌드박스 준비**: 에이전트 코드 통제를 위해 Deno 대신 OS에 격리 시스템 명령어로 작용하는 **`uv --isolated` 환경** 구동법 숙지. 네트워크를 타지 않게 확인.

### Step 4: 통일된 체계 생성 (Repository)
*   **상태**: [완료됨 - `backend/`, `frontend/`, `sandbox/` 분배 완료]

## 3. 가장 시급한 넥스트 스텝 

더 이상의 논의나 설계는 불필요할 만큼 문서는 빈틈이 없습니다.
**[사용자 행동 요구 사항]**
지금 당장 로컬 OS 환경에서 `Ollama`를 받아 **`ollama run qwen2.5-coder:7b`** (혹은 llama 3.1 8B) 를 구동(Step 2)해 주십시오. 
그 속도가 5 TPS를 넘어선다면, 즉시 백엔드(Phase 1 코어 서버)의 1일 차 개발 스크립트 작성에 돌입하겠습니다.
