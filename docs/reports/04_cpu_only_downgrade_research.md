# 온디바이스 AI 에이전트 시스템: 시니어 연구원 리서치 및 기술 제안서 (CPU-Only Downgrade)

## 1. 개요 및 연구 목적
본 문서는 사용자 하드웨어 환경(NVIDIA GPU 부재, Intel 칩셋 기반 노트북)이 확인됨에 따라,존 **20B 파라미터 + VRAM 기반 아키텍처**를 전면 폐기하고 **CPU-Only 환경에서 동작 가능한 7B~8B 규모의 최적화 모델 생태계**로 다운그레이드 및 전환하기 위해 시니어 연구원 관점에서 작성된 리소스 분석 및 대체 솔루션 제안서입니다.

## 2. 하드웨어 제약 및 7B~8B 모델 리서치 (Model Selection)

인텔 랩탑 환경에서는 VRAM이 극도로 부족하므로, 시스템 메모리(RAM)를 깎아먹고 CPU 코어 연산(AVX2/AVX-512 명령어 세트)에 의존해야 합니다. 따라서 모델의 크기를 절반 이하(7B~8B)로 줄이는 대신 **단위 파라미터 당 코딩(Logic/Reasoning) 지능이 극단적으로 높은 모델**을 수배해야 합니다.

### 추천 모델 1순위: `Qwen2.5-7B-Coder-Instruct`
*   **리서치 결과**: 현재 7B 체급 중 코딩 능력과 에이전트 활용 측면에서 압도적인 1위 모델입니다. `SmolAgents`와 같은 Code-Agent 로직을 구동할 때 20B 범용 모델과 맞먹는 파이썬 코드 생성 능력을 보여줍니다.
*   **리소스 (Q4_K_M 양자화 시)**: 약 4.5GB RAM만 점유합니다. 인텔 i5/i7급 랩탑에서 초당 약 10~15 토큰의 쾌적한 속도를 보장합니다.

### 추천 모델 2순위: `Llama-3.1-8B-Instruct`
*   **리서치 결과**: 한국어 지시 따르기와 일반적인 RAG 질문/답변에 있어서 가장 육각형 밸런스가 좋은 모델입니다.
*   **리소스 (Q4_K_M 양자화 시)**: 약 4.7GB RAM 점유.

### 추천 모델 3순위: `Phi-3.5-mini-instruct` (3.8B 초소형)
*   **리서치 결과**: 만약 7B 모델조차 노트북에서 버벅인다면 최종적으로 선택할 수 있는 마이크로소프트의 마스터피스입니다. 작지만 Code-Agent 뼈대를 억지로 굴러가게 할 수 있습니다. RAM 점유율 2.5GB.

## 3. CPU-Only 환경의 아키텍처 전환 전략 (Optimization Strategy)

### 3.1 병목 해소: Ollama CPU 스레드 최적화
*   GPU가 없으므로 `llama.cpp` 백엔드를 쓰는 Ollama는 오직 CPU 스레드로 연산합니다. 
*   **설정 변경 사항**: Ollama 구동 시 환경 변수로 `OLLAMA_NUM_PARALLEL=1` (다중 유저 동시 접속 차단) 및 시스템의 물리 코어 수에 맞추어 `num_thread`를 LLM 파라미터에 명시적으로 고정하여 CPU 과열(스로틀링)을 방어합니다.

### 3.2 LightRAG 및 임베딩 부담 경감
*   **문제점**: 20B 기반에서는 LightRAG의 문서 요약/엔티티 추출 연산이 수 분 이내로 끝났으나, CPU 7B 모델로 문서를 인제스천(Ingestion)하면 노트북 팬이 심각하게 돌며 수십 분이 걸립니다.
*   **대체제 (안전장치)**: LightRAG 옵션 중 엔티티 추출 강도를 낮추거나, 임베딩 모델 자체를 `BAAI/bge-m3` 대신 훨씬 가벼운 `jhgan/ko-sroberta-multitask` (100MB대) 로 교체해야 합니다.

### 3.3 UI 하드웨어 모니터링 타겟 변경
*   프론트엔드 관제 보드의 지표가 `VRAM 게이지`에서 **`System RAM 및 CPU 사용률(%) 게이지`**로 완전히 대체되어야 합니다. CPU 점유율이 90%를 넘나드는 현상을 상시 관찰하여 노트북 과열을 체크해야 합니다.

## 4. 결론 및 종합 제안
NVIDIA GPU가 없는 랩탑 환경이라는 **치명적 제약** 속에서도 쾌적한 온디바이스 에이전트를 영위하기 위해선 "선택과 집중"이 필요합니다. 
시니어 연구원의 권고로서, 메인 두뇌는 **`Qwen2.5-7B-Coder` (또는 Llama-3.1-8B)** 4비트 양자화 모델로 교체하고, 인프라스트럭처의 모든 병목 모니터링을 "VRAM"에서 **"CPU 스레드와 System RAM"** 관리로 전환하여 전체 설계도(블루프린트 등)를 긴급 수정하도록 조치하겠습니다.
